#### 2. `write_center()` and `write_right()` use byte count instead of display width

- **File:** `src/ui/renderer.hpp:186,197`
- **Source:** [review, correctness, performance, undirected] × 3 agents
- **Issue:** Both methods use `text.size()` (byte count) for centering/alignment math. For `"日本語"` (9 bytes, 6 display cells), centering within width 20 gives offset 5 instead of correct offset 7.
- **Impact:** Any non-ASCII text centered or right-aligned is visually misaligned. Truncation also cuts at wrong byte boundary, potentially producing invalid UTF-8.
- **Suggested fix:**
  ```cpp
  // write_center:
  int display_len = static_cast<int>(display_width(text));
  if (display_len >= width) {
      write(base_x, y, truncate(text, width), style);
      return;
  }
  // write_right: same pattern
  ```
- **Severity:** Critical

#### 3. `truncate()` missing CJK ranges from `display_width()`

- **File:** `src/core/string_utils.hpp:89-98`
- **Source:** [review, correctness]
- **Issue:** `truncate()` checks 10 CJK ranges but `display_width()` checks 16. Missing in truncate: CJK Radicals (0x2E80–0x2EFF), CJK Strokes (0x31C0–0x31EF). Also the merged range `0x20000-0x2CEAF` in truncate covers Extensions B–E, but display_width has them as separate ranges.
- **Impact:** Strings containing CJK Radicals or Strokes characters will be truncated at the wrong display width, causing text overflow in list items, labels, and panel titles.
- **Suggested fix:** Extract a shared `codepoint_width(char32_t)` helper function used by both `display_width()` and `truncate()`.
- **Severity:** Critical

#### 4. SIGWINCH handler can use-after-free on terminal destruction

- **File:** `src/platform/terminal_posix.cpp:234-238` (sigwinch_handler), `32-35` (destructor)
- **Source:** [review, correctness]
- **Issue:** The signal handler dereferences `g_active_terminal` and writes to `resize_pending_`. If SIGWINCH fires during terminal destruction (between the null check and the write), it accesses freed memory. The destructor sets `g_active_terminal = nullptr` but doesn't block signals during the transition.
- **Impact:** Undefined behavior (use-after-free) on resize during teardown. Can cause crashes or memory corruption during abnormal shutdowns.
- **Suggested fix:** Use a separate `static volatile sig_atomic_t g_resize_pending` global flag. The signal handler sets the flag; the main loop checks and clears it. Remove `g_active_terminal` and instance-level `resize_pending_`.
- **Severity:** Critical

#### 5. `EventType::Quit` not handled → 100% CPU spin on EOF

- **File:** `src/main.cpp:140-255` (handle_event)
- **Source:** [review, undirected]
- **Issue:** When stdin reaches EOF (terminal closed, pipe broken), `PosixInput::poll()` returns `Event(EventType::Quit)`. But `handle_event()` only handles `KeyPress` and `Resize` events. `Quit` falls through to `dispatch_event()` and returns `true`, causing the inner `while (auto event = poll_input())` loop to continue forever. `select()` keeps reporting EOF as readable.
- **Impact:** Application spins at 100% CPU when terminal is closed or stdin is redirected from an empty file.
- **Suggested fix:** Add explicit Quit handling:
  ```cpp
  if (e.type == EventType::Quit) {
      g_running = false;
      return false;
  }
  ```
- **Severity:** Critical

#### 6. Alt+Tab wrap-around has no visibility check

- **File:** `src/main.cpp:200-201`
- **Source:** [review, undirected]
- **Issue:** When Alt+Tab cycles past the last window and wraps to `active->windows().front()`, there is no visibility check. If the first window is minimized or hidden, focus goes to an invisible window with no visual feedback.
- **Impact:** User appears "stuck" — cycling windows does nothing visible because the target is hidden.
- **Suggested fix:**
  ```cpp
  if (!next_win) {
      for (auto& win : active->windows()) {
          if (win->visible()) { next_win = win.get(); break; }
      }
  }
  ```
- **Severity:** Critical

#### 7. Escape key captured unconditionally — starves focused widgets

- **File:** `src/main.cpp:215`
- **Source:** [review, undirected]
- **Issue:** `if (e.key_code == Keys::Escape)` has no modifier check. Every Escape press minimizes the focused window before `dispatch_event()` is called. A focused List, dropdown, or modal dialog can never receive Escape for its own purposes.
- **Impact:** Any widget needing Escape for cancel/close functionality is broken.
- **Suggested fix:** Require no modifiers: `if (e.key_code == Keys::Escape && !e.mods.alt && !e.mods.control && !e.mods.shift)`. Or dispatch to focused widget first and only minimize if unhandled.
- **Severity:** Critical

#### 8. `remove_desktop` moves windows to itself when removing active desktop

- **File:** `src/desktop/desktop_manager.hpp:52-57`
- **Source:** [review, undirected]
- **Issue:** When removing the active desktop: `auto* active = active_desktop()` points to the desktop being removed. Then `for (auto& win : (*it)->windows()) active->add_window(win)` moves windows from the desktop TO itself — doubling the `windows_` vector. After `desktops_.erase(it)`, `active` is a dangling pointer.
- **Impact:** Wasted CPU cycles (doubling vector entries that are immediately destroyed). Dangling raw pointer is a latent time-bomb — any logging or access between erase and reassign would crash.
- **Suggested fix:** When removing the active desktop, first pick a replacement desktop, move windows there, then erase:
  ```cpp
  if (removed_index == active_index_) {
      int new_idx = std::min(removed_index, static_cast<int>(desktops_.size()) - 1);
      auto* target = desktops_[new_idx].get();
      for (auto& win : (*it)->windows()) target->add_window(win);
  }
  desktops_.erase(it);
  ```
- **Severity:** Critical

#### 9. Per-character heap allocations in `flush()` hot path

- **File:** `src/ui/renderer.hpp:252-255`
- **Source:** [review, performance]
- **Issue:** Every cell character produces a new `std::string` via `utf8_encode()`, then `+=` on `run` may reallocate. On a 200×50 terminal with 50% dirty rows: ~5,000 heap allocations per frame × 20fps = ~100,000 allocs/sec.
- **Impact:** High CPU overhead from malloc/free churn. Measurable latency spikes during animation.
- **Suggested fix:** Provide output buffer pointer to `utf8_encode` so it writes directly into a pre-allocated buffer:
  ```cpp
  static int utf8_encode(char32_t ch, char* out) {
      if (ch < 0x80) { out[0] = (char)ch; return 1; }
      // ... write bytes, return count
  }
  ```
- **Severity:** Critical

---

#### 10. `Desktop::windows()` returns by value — copies entire vector on every call

- **File:** `src/desktop/desktop.hpp:52-55`
- **Source:** [review, performance, undirected]
- **Issue:** Returns `std::vector<std::shared_ptr<Window>>` by value. Called from: render loop (every frame), event dispatch, desktop switching, resize handling. Each call allocates a new vector and copies all shared_ptrs with refcount bumps.
- **Impact:** With 5 windows and 10 calls/frame: 50 allocations and 50 atomic increments per frame.
- **Suggested fix:**
  ```cpp
  const std::vector<std::shared_ptr<Window>>& windows() const { return windows_; }
  ```
- **Severity:** Critical

#### 11. `utf8_decode` accepts overlong encodings, surrogates, and out-of-range codepoints

- **File:** `src/core/string_utils.hpp:15-30`
- **Source:** [review, correctness]
- **Issue:** No validation of minimal encoding length, surrogate halves (U+D800–U+DFFF), or codepoints > U+10FFFF.

#### 12. `utf8_encode` produces invalid UTF-8 for invalid codepoints

## Pattern-Aggregated Findings

### Pattern: Byte count vs display width misuse (4 occurrences)
**Locations:**
1. `renderer.hpp:186` — `write_center` uses `text.size()`
2. `renderer.hpp:197` — `write_right` uses `text.size()`
3. `string_utils.hpp:89-98` — `truncate` missing CJK ranges vs `display_width`
4. `main.cpp:192` — top bar `repeat()` uses `bar_text.size()`

**Root cause:** No shared convention for "use display_width for layout, text.size() for memory". String utility functions correctly use `display_width()`, but Renderer methods duplicate the logic incorrectly.

**Suggested fix:** Add `display_width()` assertions or helper methods for all layout calculations. Audit all uses of `text.size()` in UI code.

## Verdict

**Request changes** — 10 Critical findings must be addressed before next release.

The codebase is architecturally sound (clear layering, good platform abstraction, effective double-buffered renderer). However, there are real bugs that cause incorrect behavior (UTF-8 display width, Quit event handling, window visibility) and a compilation failure (`tui.hpp` broken include).

### Priority Fix List (v0.2.1)

1. Remove `#include "window/window_manager.hpp"` from `tui.hpp` (1 line)
2. Fix `write_center`/`write_right` to use `display_width()` (2 lines)
3. Unify CJK ranges between `truncate()` and `display_width()` (shared helper)
4. Add `EventType::Quit` handler in `handle_event()` (4 lines)
5. Fix Alt+Tab visibility check in wrap-around (3 lines)
6. Initialize all variables in `input_posix.cpp` (5 lines)
7. Add negative dimension guard in `resize()` (1 line)
8. Fix `remove_desktop` self-move + dangling pointer (6 lines)

### Medium Priority (v0.3.0)

- SIGWINCH handler use-after-free (architectural change)
- Per-char heap allocation in `flush()` (optimization)
- `Desktop::windows()` return const-ref (API change)
- Escape key modifier guard (UX change)
- `on_resize` dimension clamping (correctness)
- `on_resize` only active desktop (optimization)
