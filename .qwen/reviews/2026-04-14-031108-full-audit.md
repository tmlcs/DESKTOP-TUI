## Critical Findings

### C1. UTF-8 corruption pattern across 6 locations
**Files:** `renderer.hpp:79`, `renderer.hpp:158/170`, `string_utils.hpp:114-136`, `string_utils.hpp:173-188`, `list.hpp:51-53`, `panel.hpp:68-70`
- `Renderer::write()` iterates `for (char c : text)` byte-by-byte, splitting multi-byte UTF-8 codepoints across cells
- `flush()` re-encodes each byte as standalone UTF-8 → garbage output
- Same bug in `pad`/`center`/`right_align` (truncates mid-codepoint → invalid UTF-8)
- Same bug in `word_wrap` (wraps by byte count, not display width)
- Same bug in `List::render()` and `Panel` title centering

### C2. `DesktopManager` constructor leaves `active_` null
**File:** `desktop_manager.hpp:24-30`
- App launches broken: welcome message never renders, status bar empty, input dropped
- Only "unbreaks" when user presses Alt+1..9 or Alt+arrows

### C3. `remove_desktop()` corrupts `active_index_`
**File:** `desktop_manager.hpp:57-63`
- Removing desktop before active one shifts indices but `active_index_` never decremented
- Indicator highlights wrong dot, `get_desktop()` returns wrong desktop

### C4. Dangling `active_` pointer when removing active desktop
**File:** `desktop_manager.hpp:47-61`
- `active_` (raw pointer) never updated when active desktop is removed
- Crash on subsequent `active_desktop()` dereference

### C5. Escape key permanently destroys window access
**File:** `main.cpp:194-206`
- `minimize()` → invisible, no keybinding to `restore()`
- Single keystroke permanently loses window

### C6. Panel renders children without clipping
**File:** `panel.hpp:76-84`
- Children extending beyond panel content area overwrite borders and adjacent content
- Visual corruption on terminal resize

### C7. Iterator invalidation in `Signal::emit()` / `EventBus::publish()`
**Files:** `signal.hpp:30-32`, `event.hpp:109-116`
- Range-for loop invalidated if callback calls `disconnect()`
- Crash if handler unsubscribes itself during dispatch

### C8. Dead code: WindowManager, EventBus, Signal (~250 lines)
**Files:** `window_manager.hpp`, `event.hpp:111-155`, `signal.hpp`
- Never instantiated, maintenance burden

## Verdict: **Request Changes**

8 Critical findings block. Key blockers: C2 (app broken on launch), C3/C4 (state corruption), C1 (UTF-8 corruption), C5 (data-loss-level usability bug).
