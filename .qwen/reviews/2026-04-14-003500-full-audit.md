## Hallazgos Críticos

### C1. Use-after-free en `WindowManager::focus_window`
- **File:** `src/window/window_manager.hpp:80-85`
- **Source:** [review] — Agent 1 (Correctness)
- **Issue:** Tras `windows_.erase(it)`, el iterador `it` es inválido. `*it` en `push_back(*it)` es undefined behavior.
```cpp
windows_.erase(it);
windows_.push_back(*it);  // UB: iterador invalidado
```
- **Impact:** Crash o corrupción de heap al traer ventana al frente (Alt+Tab, click).
- **Fix:**
```cpp
auto ptr = std::move(*it);
windows_.erase(it);
windows_.push_back(std::move(ptr));
```

### C2. `snprintf` return value mal usado — posible write OOB
- **File:** `src/platform/terminal_posix.cpp:245`
- **Source:** [review] — Agent 1
- **Issue:** `snprintf` retorna caracteres que *se habrían escrito*, no los realmente escritos. `len` acumula valores hipotéticos que pueden exceder `buf[64]`. `buf[len - 1] = 'm'` escribe fuera del buffer.
- **Impact:** Corrupción de stack — undefined behavior.
- **Fix:** Usar retorno real de snprintf o pasar a `std::string` como ya hace `Renderer::emit_style`.

### C3. `orig_termios_` sin inicializar — terminal corrupto al salir
- **File:** `src/platform/terminal_posix.cpp:248-249`
- **Source:** [review] — Agent 4 (Undirected)
- **Issue:** Si `tcgetattr` falla en `enter_raw_mode()` (stdin es pipe, no terminal), `orig_termios_` queda con basura. `leave_raw_mode()` aplica configuración garbage corruptando la terminal.
- **Impact:** Terminal del usuario queda inutilizable — requiere `reset` manual.
- **Fix:** Inicializar `orig_termios_` en el constructor y verificar return de `tcgetattr`.

### C4. `trim()` UB en strings vacíos
- **File:** `src/core/string_utils.hpp:25-29`
- **Source:** [review] — Agent 1, Agent 2 (Quality)
- **Issue:** `do { end--; }` en string vacío decrementa past `begin()` — undefined behavior.
- **Impact:** Crash potencial.
- **Fix:** Early return para strings vacíos + loop seguro.

### C5. `write_right` calcula coordenada X incorrecta
- **File:** `src/ui/label.hpp:46`, `src/ui/renderer.hpp:171-180`
- **Source:** [review] — Agent 1
- **Issue:** `write_right(bounds_.x + bounds_.w - 1, ..., bounds_.w)` suma `bounds_.w` dos veces: `start_x = (bounds_.x + bounds_.w - 1) + bounds_.w - len`.
- **Impact:** Texto alineado a derecha renderiza fuera de posición.
- **Fix:** Pasar `bounds_.x` en vez de `bounds_.x + bounds_.w - 1`.

### C6. `write_center` ignora widget x-offset
- **File:** `src/ui/label.hpp:44`, `src/ui/renderer.hpp:161-168`
- **Source:** [review] — Agent 1
- **Issue:** Centra respecto a columna 0, no respecto al bounds del widget.
- **Impact:** Texto centrado desalineado en widgets con x > 0.
- **Fix:** Agregar parámetro `base_x` a `write_center`.

### C7. Generic platform bloquea indefinidamente
- **File:** `src/platform/input_generic.cpp:21`
- **Source:** [review] — Agent 4
- **Issue:** `has_input()` siempre retorna `true`. `poll()` llama `getchar()` que bloquea. El main loop nunca llega al sleep de 16ms.
- **Impact:** App congelada al iniciar en plataforma genérica.
- **Fix:** Implementar `select()`/`poll()` no-bloqueante o retornar `false`.

### C8. Panel no clippea hijos — overflow visual
- **File:** `src/ui/panel.hpp:70-78`
- **Source:** [review] — Agent 4
- **Issue:** `content_area()` se calcula pero nunca se usa para clippear. Hijos renderizan fuera de los bordes del panel.
- **Impact:** Widgets se superponen a elementos adyacentes.
- **Fix:** Intersecar child bounds con content_area antes de renderizar.

### C9. `std::codecvt` deprecated — build break futuro
- **File:** `src/core/string_utils.hpp:7`
- **Source:** [review] — Agent 1, Agent 2
- **Issue:** `std::codecvt_utf8_utf16` deprecated en C++17, removido en C++26.
- **Impact:** Compilación falla en C++26 o compiladores estrictos.
- **Fix:** Implementar decoder UTF-8 manual (~20 líneas).

### C10. Windows `draw_hline` produce garbage con VT mode
- **File:** `src/platform/terminal_win.cpp:141-143`
- **Source:** [review] — Agent 2
- **Issue:** `std::string(w, '\xe2\x94\x80')` trunca a `'\x80'` — solo 1 byte de 3-byte UTF-8.
- **Impact:** Línea horizontal muestra caracteres basura.
- **Fix:** Concatenar string de 3 bytes en loop como hace la versión POSIX.

### C11. Windows `fill()` ignora parámetro style
- **File:** `src/platform/terminal_win.cpp:121-133`
- **Source:** [review] — Agent 2
- **Issue:** En modo no-VT, `FillConsoleOutputCharacterA` no aplica color/estilo.
- **Impact:** Fill con colores incorrectos en Windows sin VT.
- **Fix:** Llamar `FillConsoleOutputAttribute` con el estilo apropiado.

### C12. 100% CPU en idle — render incondicional cada 16ms
- **File:** `src/main.cpp:96-104`
- **Source:** [review] — Agent 3 (Performance)
- **Issue:** `render()` se llama siempre. `clear()` hace O(N) loop y setea `dirty_=true`. 2M comparaciones/frame a 60fps = 120M ops/s en idle.
- **Impact:** Un core de CPU al 100% incluso sin interacción.
- **Fix:** Solo render si hubo input o dirty flag. Sleep largo (100ms) en idle.

## Hallazgos — Suggestion

### S1. `utf8_encode` aloca heap por carácter en flush hot path
- **File:** `src/ui/renderer.hpp:248-263`
- **Source:** [review] — Agent 3
- **Issue:** Cada celda genera un `std::string` heap-allocated. En 1920×1080: ~2M allocs/frame.
- **Fix:** Función `utf8_encode_append(char32_t, std::string&)` que usa `push_back`.

### S2. `emit_style` en Renderer usa `to_string` + `+=`
- **File:** `src/ui/renderer.hpp:224-244`
- **Source:** [review] — Agent 3
- **Issue:** 3× `std::to_string` allocations por color TrueColor. Contraste con POSIX que usa `snprintf` en buffer stack.
- **Fix:** Usar `snprintf` en buffer stack como `PosixTerminal::emit_style`.

### S3. `EventBus` y `Signal` son dead code completo
- **File:** `src/core/event.hpp:108+`, `src/core/signal.hpp`
- **Source:** [review] — Agent 2
- **Issue:** ~100 líneas de código que nunca se instancian ni se usan en ningún punto del proyecto.
- **Fix:** Eliminar o documentar como "reserved for future use".

### S4. ~15 métodos públicos nunca llamados
- **Files:** Múltiples
- **Source:** [review] — Agent 2
- **Issue:** `cursor_save/restore`, `sync`, `toggle_minimize`, `set_focusable`, `move_to_desktop`, `close_all`, `focus_next/prev`, `minimize_window`, `find_window`, `remove_desktop`, `find_desktop`, `get_desktop`, `DefaultColor`, `with_fg/bg/bold/reverse`, `write_at`.
- **Impact:** API bloat, falsa sensación de funcionalidad probada.
- **Fix:** Eliminar o agregar tests de integración.

### S5. `draw_rect` y `emit_style` duplicados 3 veces
- **Files:** `terminal_posix.cpp`, `terminal_win.cpp`, `terminal_generic.cpp`
- **Source:** [review] — Agent 2
- **Issue:** ~50 líneas idénticas en 3 archivos. Bug fix requiere triple aplicación.
- **Fix:** Extraer a función compartida con callback de capability check.

### S6. Input buffer puede crecer sin límite
- **File:** `src/platform/input_posix.cpp:47-55`
- **Source:** [review] — Agent 1
- **Issue:** Secuencias malformed crecen buffer ~16 bytes/poll. A 60fps: ~1KB/s.
- **Fix:** MAX_BUFFER = 256 bytes. Clear si se excede.

### S7. `Renderer::resize()` acepta dimensiones negativas
- **File:** `src/ui/renderer.hpp:36-44`
- **Source:** [review] — Agent 1
- **Issue:** `cols * rows` como `size_t` con negativos = wrap a valor enorme = `bad_alloc`.
- **Fix:** Early return si `cols <= 0 || rows <= 0`.

### S8. `g_active_terminal` dangling pointer tras destrucción
- **File:** `src/platform/terminal_posix.cpp:252-261`
- **Source:** [review] — Agent 1
- **Issue:** No se limpia en destructor. SIGWINCH post-destrucción = write to freed memory.
- **Fix:** `if (g_active_terminal == this) g_active_terminal = nullptr;` en destructor.

### S9. WindowManager y DesktopManager — contradicción arquitectónica
- **Files:** `window_manager.hpp`, `desktop_manager.hpp`
- **Source:** [review] — Agent 4
- **Issue:** Dos sistemas independientes de ownership de ventanas. `WindowManager` está completamente unused (dead code). `DesktopManager` hace su propio lifecycle.
- **Fix:** Eliminar `WindowManager` o integrarlo como backend de `DesktopManager`.

### S10. `Desktop::move_window_to_desktop` — copia vector + iteración redundante
- **File:** `src/desktop/desktop_manager.hpp:111-125`
- **Source:** [review] — Agent 4
- **Issue:** `d->windows()` retorna copia. `find_window` + loop sobre copia = redundante.
- **Fix:** Obtener shared_ptr una vez, remover, agregar al target.

### S11. `WindowsInput::hIn_` nunca validado
- **File:** `src/platform/input_win.cpp:109`
- **Source:** [review] — Agent 4
- **Issue:** `GetStdHandle` puede retornar `INVALID_HANDLE_VALUE` (redirección, GUI). Sin validación.
- **Fix:** Validar handle en constructor.

### S12. Señales instaladas antes de init terminal
- **File:** `src/main.cpp:48-54`
- **Source:** [review] — Agent 4
- **Issue:** `signal(SIGINT)` antes de `term_->init()`. atexit handler corre después de destructor.
- **Fix:** Mover instalación de señales después de init exitoso.

### S13. Panel/Widget — sistema de coordenadas inconsistente
- **File:** `src/ui/panel.hpp:114`
- **Source:** [review] — Agent 4
- **Issue:** `content_area()` sugiere coordenadas locales pero todo usa coordenadas absolutas de pantalla.
- **Fix:** Implementar transformación local-a-pantalla o remover `content_area()`.

### S14. `WindowsTerminal::enter_raw_mode` — modo console set twice
- **File:** `src/platform/terminal_win.cpp:157-166`
- **Source:** [review] — Agent 4
- **Issue:** Segunda llamada a `SetConsoleMode` sobrescribe flags de la primera.
- **Fix:** Combinar flags en una sola llamada.

### S15. `EventBus::publish` — iterator invalidation
- **File:** `src/core/event.hpp:109-117`
- **Source:** [review] — Agent 4
- **Issue:** Handler que llama `unsubscribe()` invalida iteradores del range-for.
- **Fix:** Copiar vector antes de iterar o deferred-unsubscribe.

### S16. Falso evento Escape en input POSIX fragmentado
- **File:** `src/platform/input_posix.cpp:71-76`
- **Source:** [review] — Agent 4
- **Issue:** Secuencias escape incompletas retornan `Keys::Escape` → minimize window espurio.
- **Fix:** Retornar `nullopt` y limpiar buffer en siguiente poll cycle.

### S17. `List::render` trunca por bytes, no display width
- **File:** `src/ui/list.hpp:45-47`
- **Source:** [review] — Agent 4
- **Issue:** `display.size()` (bytes) vs `bounds_.w` (celdas). UTF-8 multibyte se trunca mal.
- **Fix:** Usar `display_width()` para comparación.

### S18. `truncate()` compara codepoints con display width
- **File:** `src/core/string_utils.hpp:100-103`
- **Source:** [review] — Agent 1
- **Issue:** `str.size()` (codepoints) vs `max_width` (celdas de display). CJK overflow.
- **Fix:** Usar `display_width()` en comparación + truncar por display width.

### S19. `write_styled` vs `Renderer::emit_style` — dos sistemas de estilo
- **File:** `src/platform/terminal.hpp:58`, `src/ui/renderer.hpp:219`
- **Source:** [review] — Agent 4
- **Issue:** `terminal->write_styled()` resetea todos los atributos con `\033[0m`. Si se mezcla con Renderer, corrompe el tracking de estilo del renderer.
- **Fix:** Documentar incompatibilidad o hacer terminal writes private cuando hay Renderer activo.

### S20. `unsubscribe` escanea todos los tipos de evento O(N×M)
- **File:** `src/core/event.hpp:98-105`
- **Source:** [review] — Agent 1
- **Issue:** Busca subscription ID en TODOS los tipos de evento cuando se registró en uno solo.
- **Fix:** Map<SubscriptionId, EventType> para lookup directo.

### S21. Falta `[[nodiscard]]` en factory functions
- **File:** `src/platform/terminal.hpp:98-99`, `input.hpp:30-31`
- **Source:** [review] — Agent 2
- **Fix:** `[[nodiscard]] ITerminal* create_terminal();`

### S22. Windows `emit_style` faltan flags blink y hidden
- **File:** `src/platform/terminal_win.cpp:247-252`
- **Source:** [review] — Agent 2
- **Issue:** POSIX tiene 7 flags, Windows solo 5.
- **Fix:** Agregar `blink` y `hidden`.
